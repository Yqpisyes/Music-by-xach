Markdown 书写指南
Markdown 是一种轻量级标记语言，旨在以易读易写的纯文本格式编写文档，并可以转换为有效的 HTML。本指南将介绍最常用的 Markdown 语法，助你快速上手。

扩展与兼容性

标题
使用 # 表示标题，# 的数量代表标题级别（1 到 6 级）。

markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
效果：

一级标题
二级标题
三级标题
四级标题
五级标题
六级标题
段落与换行
段落由一行或多行文本组成，段落之间用一个空行分隔。
如果要在段落内换行，在行尾添加两个空格然后回车。

markdown
这是第一段。  
这是同一段内的第二行（行尾有两个空格）。

这是第二段。
强调
使用 * 或 _ 包围文本实现斜体或粗体。

语法	效果
*斜体* 或 _斜体_	斜体
**粗体** 或 __粗体__	粗体
***粗斜体***	粗斜体
~~删除线~~	~~删除线~~
注意：符号与文本之间不要有空格。

列表
无序列表
使用 -、* 或 + 作为列表标记。

markdown
- 项目一
- 项目二
  - 子项目（缩进 2 或 4 空格）
- 项目三
效果：

项目一

项目二

子项目

项目三

有序列表
使用数字加点号。

markdown
1. 第一项
2. 第二项
   1. 子项 1
   2. 子项 2
3. 第三项
效果：

第一项

第二项

子项 1

子项 2

第三项

链接
行内式
markdown
[Google](https://www.google.com)
[带标题的链接](https://example.com "这是标题")
效果：Google

参考式
先定义链接，然后在文中引用。

markdown
这是一个 [参考式链接][1] 的例子。

[1]: https://example.com "可选标题"
自动链接
直接使用 < > 包裹 URL 或邮箱。

markdown
<https://www.example.com>
<email@example.com>
图片
语法与链接类似，前面加 !。

markdown
![替代文本](图片URL "可选标题")
示例：

markdown
![Markdown 图标](https://markdown-here.com/img/icon256.png)
图片无法显示时，替代文本会显示。

引用
使用 > 表示引用，可以嵌套。

markdown
> 这是一段引用。
> 可以跨多行。
>
> > 嵌套引用
效果：

这是一段引用。
可以跨多行。

嵌套引用

代码
行内代码
用反引号 ` 包裹。

markdown
请执行 `git status` 命令。
代码块
用三个反引号 ``` 包裹，并可选指定语言以启用语法高亮。

markdown
```python
def hello():
    print("Hello, Markdown!")
```
效果：

python
def hello():
    print("Hello, Markdown!")
也可以使用缩进四个空格或一个制表符表示代码块，但不支持语法高亮。

分割线
使用三个或更多 -、* 或 _ 单独成行。

markdown

表格
使用管道符 | 分隔列，第二行指定对齐方式（- 左对齐，:-: 居中，-: 右对齐）。

markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 单元格 | 单元格   | 单元格 |
| 单元格 | 单元格   | 单元格 |
效果：

左对齐	居中对齐	右对齐
单元格	单元格	单元格
单元格	单元格	单元格
任务列表
在任务项前加 - [ ]（未完成）或 - [x]（已完成）。

markdown
- [x] 已完成任务
- [ ] 未完成任务
  - [ ] 子任务
效果：

已完成任务

未完成任务

子任务

脚注
在文本中添加标记 [^标记]，然后在文档末尾定义脚注内容。

markdown
这是一段需要注释的文字[^1]。

[^1]: 这是脚注的内容，可以写多行。
效果（取决于渲染器，通常显示为上标并可点击跳转）。

转义字符
如果希望显示原本有特殊含义的字符（如 *、#），在字符前加反斜杠 \。

markdown
\*这不是强调\*
\# 这不是标题
内嵌 HTML
Markdown 兼容 HTML，可以直接在文档中使用 HTML 标签，实现更复杂的排版（如改变颜色、大小等）。

markdown
这是一段 <span style="color:red">红色文字</span>。
扩展与兼容性
不同平台（如 GitHub、Typora、Obsidian）可能对 Markdown 有各自的扩展语法（如数学公式、图表、高亮等）。使用时请查阅对应平台的文档。

GitHub Flavored Markdown (GFM)：支持任务列表、表格、删除线等。

数学公式：常用 $ 或 $$ 包裹 LaTeX 公式（需平台支持）。

图表：Mermaid、PlantUML 等（需平台支持）。

以上是 Markdown 的常用语法，多加练习即可熟练运用。Happy writing!
